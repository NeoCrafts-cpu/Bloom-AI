# Bloom AI — Demo Verification Guide

Use this checklist to verify the app works end-to-end for judging.

## Prerequisites

- MetaMask with ValueChain Testnet (Chain ID `138565`)
- SoDEX testnet account with USDC (for live execution)
- API keys configured on backend (see README)

## Step 1: API Health

```bash
curl https://bloom-ai-mqrb.onrender.com/health
```

Expected:
- `integrations.sodex.ok`: `true` (tickers + klines reachable)
- `integrations.sosovalue.ok`: `true` when `SOSOVALUE_API_KEY` is set
- `integrations.priceSource`: `"sodex"` when market data is live
- `executionMode`: `"live"` if `SODEX_API_PRIVATE_KEY` set, else `"simulated"`

## Step 2: SoSoValue Integration

```bash
curl https://bloom-ai-mqrb.onrender.com/api/market/etf-flows
curl https://bloom-ai-mqrb.onrender.com/api/market/sentiment?limit=5
```

Expected: JSON envelopes with `meta.status` and data when API key is configured.

## Step 3: SoDEX Integration

```bash
curl "https://bloom-ai-mqrb.onrender.com/api/market/klines/BTC?interval=1h&limit=24"
curl https://bloom-ai-mqrb.onrender.com/api/market/sodex/orderbook/vBTC_vUSDC
curl https://bloom-ai-mqrb.onrender.com/api/market/prices
```

Expected:
- Klines: non-empty OHLCV bars
- Orderbook: bids/asks arrays
- Prices: `meta.source: "sodex"` (not `"seed"`)

## Step 4: Trigger Agent Pipeline

```bash
curl -X POST https://bloom-ai-mqrb.onrender.com/api/agents/pipeline/trigger
```

Expected response includes:
- `data`: 5 agent states (journalist → chartanalyst → strategist → broker → sentinel)
- `pipeline.newsletter`: published newsletter object
- `pipeline.strategy`: generated SSI index with live asset weights
- `pipeline.sentinelDryRun.passed`: `true` for default dry-run intent
- Broker message: `"Ready for confirmation — execute via Copy Trade"`

## Step 5: Verify Generated Strategy

```bash
curl https://bloom-ai-mqrb.onrender.com/api/strategies
```

Expected: at least one pipeline-generated strategy (empty before first pipeline run).

## Step 6: Sentinel Check

```bash
curl -X POST https://bloom-ai-mqrb.onrender.com/api/sentinel/check \
  -H "Content-Type: application/json" \
  -d '{"strategyId":"<strategy-id-from-pipeline>","newsletterId":"nl-latest","userAddress":"0x0000000000000000000000000000000000000001","allocationUSD":100,"maxSlippageBps":50}'
```

Expected: `passed: true` with 8 checks listed.

## Step 7: Copy Trade (UI)

1. Open `/copy-trade?strategy=<id-from-pipeline>`
2. Connect MetaMask on ValueChain Testnet
3. Set allocation ($10–$100 recommended)
4. Sign EIP-712 authorization
5. Confirm Sentinel passes (green checks)
6. View execution result — note **SIMULATED** vs **LIVE** badge

Execution uses `POST /api/broker/execute` (not the legacy `/api/copy-trade/execute`).

## Step 8: Performance Verification

```bash
curl https://bloom-ai-mqrb.onrender.com/api/copy-trade/history
curl https://bloom-ai-mqrb.onrender.com/api/copy-trade/performance
curl https://bloom-ai-mqrb.onrender.com/api/copy-trade/audit
```

Expected after a trade:
- History contains the trade with `simulated: true/false`
- Performance shows real `totalTrades`, `totalExecutedUSD` — **no fabricated win rates**
- Audit log shows execution or sentinel_block entries

## Automated Smoke Test

```bash
# Basic checks (no pipeline — fast)
npm run smoke

# Full integration including SoSoValue + pipeline
REQUIRE_SOSOVALUE=1 RUN_PIPELINE=1 API_URL=http://localhost:4000 npm run smoke
```

## What Is NOT Mocked

- Market prices come from SoDEX (fallback: CoinGecko, never shown as live if seed)
- Strategies are generated by the Strategist agent after pipeline run
- Performance KPIs reflect verified session trades only
- Broker execution requires explicit user confirmation + valid strategy ID

## What Requires Credentials

| Feature | Requirement |
|---------|-------------|
| SoSoValue ETF/news/sentiment | `SOSOVALUE_API_KEY` |
| SoDEX market data | Public testnet (no key) |
| Live order execution | `SODEX_API_PRIVATE_KEY` + `SODEX_API_KEY_ADDRESS` |
| LLM newsletters | `OPENROUTER_API_KEY` (template fallback if missing) |
